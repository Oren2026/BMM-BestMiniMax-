## 📋 任務分類與處理

### 類型 1：簡單任務（Direct）

**定義：** 可以在一個來回內完成的任務

**處理方式：**
```
Mavis 直接處理 → 回覆用戶
```

**範例：**
- 回答問題
- 簡單資料查詢
- 格式化輸出

---

### 類型 2：中等任務（Single Worker）

**定義：** 需要多步驟但不複雜，一個 Worker 就能完成

**處理方式：**
```
Mavis 派遣 Worker → Worker 完成 → Mavis 審核 → 回覆用戶
```

**範例：**
- 寫一個頁面
- 整理一份文件
- 執行一次分析

---

### 類型 3：複雜任務（Team）

**定義：** 需要多個專業領域協作

**處理方式：**
```
Mavis 啟動團隊計劃 → 並行處理 → Verifier 驗證 → Mavis 整合 → 回覆用戶
```

**範例：**
- 建一個完整網站
- 多專案協調
- 深度研究報告

## 📁 檔案更新責任

### 任務開始時

更新 `tasks.json`：
```json
{
  "id": "T010",
  "name": "新任務",
  "agent": "Coder",
  "status": "in-progress",
  ...
}
```

### 任務完成時

1. 更新 `tasks.json` 的 status 為 `completed`
2. 更新 `news.json` 添加新聞事件
3. 更新 `api_calls.json` 的統計數字

## 🔧 更新流程

```
1. 編輯相關 JSON 檔案
       │
       ▼
2. git add + git commit
       │
       ▼
3. GIT_SSH_COMMAND="ssh -i ~/.ssh/mavis_deploy" git push
       │
       ▼
4. GitHub 更新完成，其他人可以同步
```
